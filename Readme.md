## 📚 추천 알고리즘

- 음악 추천 알고리즘은 다음과 같이 이루어집니다.
  - 특정 감정에 대해 과거 유저데이터가 있다면 과거 데이터 활용
    1. 음악데이터는 각 음악마다 태그데이터를 가지고 있음
    2. 과거 음악 데이터들을 태그기반으로 벡터화 시키고 모두 더함
    3. 2에서 만든 벡터를 유저-태그 매트릭스라고 할것
    4. 각 음악들은 태그를 가지고있기에 이것을 음악-태그 매트릭스라고 할것
    5. 유저-태그 매트릭스와 음악-태크 매트릭스를 각각 numpy의 svd 알고리즘을 활용하여 행렬분해할 것
    6. 5를 거치면 유저-잠재요인 매트릭스와 음악-잠재요인 매트릭스를 구할 수 있음
    7. 유저-잠재요인 매트릭스와 음악 잠재요인 매트릭스의 전치행렬을 내적하면 (1,음악개수)의 행렬 나옴
    8. 7에서 구한 리스트에서 가장 높은 N개의 음악을 추천함(과거 유저가 들은 음악과 특저 음악의 상관관계를 의미함)

  - 특정 감정에 대해 과거 유저데이터가 없다면 현재 유저의 유사한 유저의 데이터 활용
    1. 회원가입시 음악 취향을 받아 다중선택 가능하도록 함.
    2. 선택한것은 1, 선택하지 않은 것은 0으로 바꿔 벡터로 바꾼뒤 코사인 유사도로 유사한 유저 찾음
    3. 유사한 유저가 특정감정일 때 어떤 노래를 들었는지 확인해서 찾은 노래와 유사한 노래를 추천
    4. 유사한 노래는 TF-IDF방식을 활용함.
    5. 음악을 문서, 태그를 단어라고 생각하면 음악 태그기반의 유사한 음악을 찾는문제는 단어기반의 유사한 문서를 찾는 문제로 바뀜
    6. 하나의 단어가 단일문서에서 여러번 등장하면 TF값은 증가하고, 여러문서에서 등장하면 IDF가 감소함
    7. 이런식으로 구한 TF와 IDF를 곱해 가중치를 구하면 하나의 문서에는 여러 TF-IDF값이 들어감.
    8. 이것을 벡터화하고 코사인유사도를 이용하면 비슷한 노래를 찾을 수 있음.
    
  - 위 2가지 모두 해당되지 않으면 유저들이 가장 많이 선택한 음악 추천
    1. 유저들이 가장 많이 들었던 노래 5가지를 추천해줌.

## 📚 텍스트 분석 알고리즘

- BERT
  - BERT :  Bidirectional Encoder Representations for Transformers
  - 자연어 처리를 위한 모델로, 기존 일방향 Decoder 방식과는 달리 양방향으로 문장을 분석한다.
  - 일방향의 Decoder는 새로운 문장에 적합한 반면, BERT는 양방향으로 분석에 초점이 맞춰져 있다.

- koBERT
  - 기존 BERT 모델에 한국어 학습 신경망을 추가한 koBERT 모델을 사용.
  - koBERT를 채택한 이유 : 한국어 자연어 처리 모델 중 가장 높은 정확도를 보이고, 감정이나 긍/부정을 분석하는 프로젝트에서 널리 활용.

- Pre Training : BERT는 수백만개의 텍스트 데이터를 사전 학습
  - 문장을 Attention 행렬(문장에서의 단어 순서, 문장 시작과 끝 구별, 상황별 중요한 단어 및 문맥 이해, 오타 처리를 수치로 나타낸 행렬)로 변경
  문장 중 단어 하나를 가린 후 그 단어를 예측하는 방식으로 학습.
- Fine Tuning : 이후 이 모델에 목적에 맞는 신경망을 추가합니다.
  - 문장 + 감정 벡터 (16만개) 데이터를 학습.
  - 기쁨 0, 안정 1, ….
  - 모델이 완성된 후, 문장을 입력하면 각 감정과 얼마나 일치하는지 정도를 나타내는 비율이 도출.

- 모델 개선 관련
  - kFold를 이용하여 교차 검증(cross-validation)으로 train 데이터로의 과적합을 방지.
  - 5회에서 10회로 변경.
  - 최적의 epoch을 찾고자 노력.
  - 학습 데이터를 순차적으로 추가.

- 다른 사용자의 정원으로 이동하여 우체통을 클릭하면 해당 유저의 정원에 방명록을 남길 수 있습니다.
- 우측 상단 버튼을 클릭하여 방명록을 작성할 수 있고, 방명록의 배경 색을 선택할 수 있습니다.
- 만약 내가 이전에 작성했던 방명록을 지우고 싶다면 일기의 우측 상단에 메뉴를 눌러 삭제할 수 있습니다.
